Built for the real operator workflow

What usually happens

  • You find the Deployment, but not the warning someone dropped into Slack last week.
  • You see a scaling event, but nobody recorded why it happened.
  • You inherit an incident and the handover context is scattered across tabs.
  • You know a runbook exists, but it is not attached to the resource you are touching.

What KubeMemo changes

  • kubectl describe can show that memo context exists and how to open it.
  • Durable and runtime memos stay separate so GitOps and live ops stop fighting each other.
  • Operators can read the story in CLI cards or browse it from the memo board TUI.
  • GitOps clusters can keep durable notes in Git while runtime context still works safely in-cluster.

The product model in one glance

Durable store

Long-lived operational memory

Use durable memos for warnings, ownership, runbooks, and maintenance notes that should remain part of the resource history.

  • Backed by memos.notes.kubememo.io
  • Fits Git-managed workflows
  • Ideal for curated team knowledge
Runtime store

Temporary live cluster context

Use runtime memos for incident handover notes, expiring advisories, and activity breadcrumbs generated while the cluster is changing.

  • Backed by runtimememos.runtime.notes.kubememo.io
  • Safe for live write workflows
  • Ideal for incidents and active operations

Three-step operator loop

1

Attach durable knowledge

Leave a warning, ownership note, or rollout guidance on the resource operators will actually inspect during change or incident work.

2

Capture live context

Add temporary runtime notes during incidents and let activity capture create useful breadcrumbs as resources change over time.

3

Read it in the terminal

Use memo cards, the TUI board, and annotation discovery hints to recover context quickly when someone is under pressure.

Start with real commands

Install
Install-KubeMemo -EnableRuntimeStore -RuntimeNamespace kubememo-runtime
Create a durable memo
New-KubeMemo -Title "Orders API warm-up behavior" `
  -Summary "Expected transient 502s after deployment" `
  -Content "Ignore failures for up to 3 minutes after rollout." `
  -Kind Deployment -Namespace prod -Name orders-api `
  -NoteType warning
Capture activity
kubememo watch-activity --namespace prod --kind Deployment
Browse the board
kubememo tui
