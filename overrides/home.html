KubeMemo Operational memory for Kubernetes

Attach the missing context to the exact Kubernetes thing your operators are touching.

KubeMemo gives teams a durable memo layer for runbooks, warnings, ownership, and maintenance guidance, plus a separate runtime memo store for incidents, temporary notes, and live activity breadcrumbs.

Why it exists

KubeMemo is built for the operational gap between config, docs, and incident reality.

Docs are too far away, chat is too temporary, and Git alone does not capture the live context operators need while a cluster is changing. KubeMemo keeps that context attached to the resource, namespace, or app where it matters.

Durable knowledge

Keep runbooks and warnings with the target they belong to

Store long-lived memos for ownership, maintenance guidance, known issues, and rollout behavior.

Runtime context

Make temporary notes and incident handovers safe to write live

Use runtime memos for fast-changing context without polluting the durable record or fighting GitOps reconciliation.

Activity capture

Turn cluster changes into readable breadcrumbs

Capture meaningful workload and traffic changes as runtime memos so the next operator can see what changed and why.

Product model

Two stores, two jobs, one terminal workflow.

Durable store

Long-lived operational memory

Use durable memos for curated notes that should survive team changes and deployment cycles.

  • Warnings and expected behavior notes
  • Ownership and maintenance guidance
  • Runbooks and long-lived suppressions
  • GitOps-friendly export and import workflows
Runtime store

Live operational context

Use runtime memos for incident notes, temporary advisories, and activity breadcrumbs created while the cluster is moving.

  • Incident handovers and temporary notes
  • Expiring runtime warnings and suppressions
  • Auto-captured activity breadcrumbs
  • Safe live writes outside Git reconciliation scope

Operator workflow

Three steps from missing context to readable operational memory.

1

Attach durable knowledge

Add warnings, ownership, and runbooks to a resource, namespace, or app before the next operator needs them.

2

Capture live context

Write runtime notes during incidents and let activity capture create breadcrumbs as workloads and traffic objects change.

3

Read it where you work

Use CLI cards, the TUI board, and `kubectl describe` discovery hints to recover the story without hunting across tabs.

Use it your way

CLI, TUI, PowerShell, Homebrew, GitHub releases, and Helm.

Go CLI

Use the native KubeMemo CLI on macOS, Linux, and Windows

Run `kubememo get`, `show`, `find`, `tui`, and `watch-activity` directly from the terminal.

PowerShell wrapper

Keep PowerShell Gallery distribution for PowerShell-first teams

Use the same product surface through wrapper commands without maintaining a second implementation.

Cluster component

Enable always-on activity capture when you need it

Deploy the optional in-cluster activity watcher with Helm or installation flags for continuous breadcrumbs.

Install 
brew install kubedeckio/tap/kubememo
PowerShell 
Install-Module KubeMemo
Install-KubeMemo -EnableRuntimeStore
Browse 
kubememo tui
Activity capture 
kubememo watch-activity --namespace prod --kind Deployment
