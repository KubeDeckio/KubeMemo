Why KubeMemo exists

Without KubeMemo

  • Warnings disappear into chat threads and incident docs.
  • Manual changes are hard to explain after the fact.
  • Ownership and maintenance context lives outside the cluster.
  • Temporary notes get mixed into permanent records.

With KubeMemo

  • Resources carry discoverable memo metadata through annotations.
  • Durable and runtime notes stay separate and mode-aware.
  • GitOps clusters keep durable notes in Git and runtime notes live.
  • Operators can see the full story from CLI, TUI, or kubectl describe.

Three-step workflow

1

Install the stores

Bootstrap CRDs, optional runtime namespace, and optional RBAC with Install-KubeMemo or the Go CLI.

2

Attach context

Create durable memos for runbooks and runtime memos for live work without mixing the two models.

3

Read it where you work

Use kubememo show, Open-KubeMemoTui, or annotations on the resource to surface the right context fast.
